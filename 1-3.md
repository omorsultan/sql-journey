# SQL & Relational Databases — Study Notes

## 1. Relational Databases (SQL)
A **relational database** stores data in tables made of rows and columns, where tables can be linked to one another.

- **RDBMS** (Relational Database Management System) — software that helps create and maintain a relational database.
  - Examples: `MySQL`, `Oracle`, `PostgreSQL`, `MariaDB`

**SQL (Structured Query Language)**
- Standardized language for interacting with an RDBMS.
- Used to perform **CRUD** operations (Create, Read, Update, Delete) and admin tasks (user management, security, backup).
- Used to define tables and structures.
- ⚠️ SQL written for one RDBMS isn't always portable to another without modification.

### SQL is actually 4 languages in one

| Sub-language | Full Name | Purpose |
|---|---|---|
| **DQL** | Data Query Language | Query the database for information already stored |
| **DDL** | Data Definition Language | Define database schemas (tables, structure) |
| **DCL** | Data Control Language | Control access/permissions to data |
| **DML** | Data Manipulation Language | Insert, update, delete data |

---

## 2. Non-Relational Databases (NoSQL)
**NoSQL** = "Not only SQL." Managed by **NRDBMS** (Non-Relational DBMS), e.g. `MongoDB`, `DynamoDB`, `Apache Cassandra`, `Firebase`.

- **Implementation-specific** — no single standard language; each NRDBMS defines its own CRUD/admin syntax.

| Type | Description |
|---|---|
| **Document** | Stores records as documents (JSON, BLOB, XML, etc.) |
| **Graph** | Stores data as relational nodes and edges |
| **Key-Value Hash** | Keys mapped to values (strings, JSON, BLOB, etc.) |

---

## 3. Database Queries
A **query** is a set of instructions (written in SQL) given to the RDBMS that tells it exactly what information to retrieve.

- Databases can hold tons of data hidden in complex schemas.
- Goal: retrieve only the specific data you need (like a Google search, but for your DB).

```sql
SELECT employee.name, employee.age
FROM employee
WHERE employee.salary > 3000;
```

---

## 4. Keys

| Key Type | Definition |
|---|---|
| **Primary Key** | Uniquely identifies each row in a table (e.g. `student_id`) |
| **Surrogate Key** | A key with no mapping to anything in the real world (purely for identification) |
| **Foreign Key** | An attribute stored in a table that stores the primary key of a row in *another* table — used to link/relate two tables |
| **Composite Key** | A primary key made up of **two or more** attributes together |

**Example schema relationships:**
- `Employee.branch_id` → Foreign Key referencing `Branch.branch_id`
- `Works_With(emp_id, client_id)` → Composite Key (both columns together uniquely identify a row)

## 6. Common Data Types

| Type | Description |
|---|---|
| `INT` | Whole numbers |
| `DECIMAL(M,D)` | Exact decimal value (M = total digits, D = digits after decimal) |
| `VARCHAR(L)` | String of text, max length L |
| `BLOB` | Binary Large Object — stores large binary data |
| `DATE` | Format: `YYYY-MM-DD` |
| `TIMESTAMP` | Format: `YYYY-MM-DD HH:MM:SS` — used for recording date+time |

## 7. Creating & Modifying Tables (DDL)

**Create a database:**
```sql
CREATE DATABASE giraffe;
```

**Create a table:**
```sql
CREATE TABLE student (
    student_id INT,
    name VARCHAR(20),
    major VARCHAR(20),
    PRIMARY KEY(student_id)
);
```

**Inspect a table's structure:**
```sql
DESCRIBE student;
```

**Add a column to an existing table:**
```sql
ALTER TABLE student ADD gpa DECIMAL(3,2);
```

**Delete an entire table:**
```sql
DROP TABLE student;
```

**Another example table:**
```sql
CREATE TABLE user (
    user_id INT,
    user_name VARCHAR(20),
    role VARCHAR(20),
    PRIMARY KEY (user_id)
);

ALTER TABLE user ADD salary DECIMAL(5);
```

### Column Constraints

| Constraint | Effect |
|---|---|
| `NOT NULL` | Column cannot contain `NULL` values |
| `UNIQUE` | Every value in the column must be unique across rows |
| `DEFAULT 'value'` | Sets a fallback value if none is provided (e.g. `DEFAULT 'undecided'`) |
| `AUTO_INCREMENT` | Value is automatically incremented each time a new row is added |

```sql
CREATE TABLE student (
    student_id INT AUTO_INCREMENT,
    name VARCHAR(20) NOT NULL,
    major VARCHAR(20) UNIQUE DEFAULT 'undecided',
    PRIMARY KEY(student_id)
);
```

## 8. Inserting & Selecting Data (DML / DQL)

**Insert a full row:**
```sql
INSERT INTO student VALUES (1, 'Jack', 'Biology');
```

**Insert a partial row** (omitted columns become `NULL`, unless a `DEFAULT` is set):
```sql
INSERT INTO student (student_id, name) VALUES (3, 'Claire');
```

**Select all rows/columns:**
```sql
SELECT * FROM student;
```

**Select specific columns:**
```sql
SELECT student.name, student.major FROM student;
```

### Sorting & Limiting

```sql
-- Sort by name, descending
SELECT name, major FROM student ORDER BY name DESC;

-- Sort by id, descending, only return top 2 rows
SELECT * FROM student ORDER BY student_id DESC LIMIT 2;
```

### Filtering with WHERE

```sql
-- Equals
SELECT * FROM student WHERE major = 'Biology';

-- OR
SELECT name, major FROM student WHERE major = 'Chemistry' OR name = 'Kate';

-- Not equal
SELECT name, major FROM student WHERE major <> 'Chemistry';
```

**Comparison/logical operators:** `<`, `>`, `<=`, `>=`, `=`, `<>`, `AND`, `OR`

---

## Quick Reference Summary

- **SQL** = standardized language for **relational** DBs (4 sub-languages: DQL, DDL, DCL, DML)
- **NoSQL** = umbrella term for non-relational DBs (document, graph, key-value)
- **Query** = instructions to retrieve specific data from a DB
- **Keys**: Primary (unique row ID), Foreign (links tables), Surrogate (no real-world meaning), Composite (multi-column key)
- **DDL**: `CREATE`, `ALTER`, `DROP`
- **DML/DQL**: `INSERT`, `SELECT`, `WHERE`, `ORDER BY`, `LIMIT`
- **Constraints**: `NOT NULL`, `UNIQUE`, `DEFAULT`, `AUTO_INCREMENT`
